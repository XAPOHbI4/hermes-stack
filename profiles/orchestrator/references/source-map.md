# Source map & trust pyramid

Где брать правду и кому верить, если источники спорят. Грузится по триггеру «выбор источника / приоритет слоёв».

## Trust pyramid (приоритет при конфликте — сверху вниз)
Если слои противоречат — НЕ выбирай удобный. Проверяй сверху вниз:
```
rules (system/CLAUDE.md) → project contract (AGENTS.md + references/) →
source of truth (wiki/Kanban/Notion) → live state (systemctl/hermes/БД прямо сейчас) →
references → skills → reports/receipts → profile memory → old chats (только recall)
```
Главное: **старый диалог и память НЕ побеждают live state и source of truth.** Память — стабильные предпочтения, не факты о текущем состоянии. Перед утверждением о состоянии системы — `live check`, а не «помню из прошлой сессии».

## Source cards

### Kanban boards (`/root/.hermes/kanban/boards/<board>/`)
- **Покрывает:** состояние задач, кто/что/статус, proof закрытия, история (`task_runs`, `task_events`).
- **НЕ покрывает:** почему принято арх-решение (это в контракте/wiki), личные задачи (это Notion).
- **Trust:** HIGH — durable state machine, source of truth по задачам.
- **Freshness:** реальное время (dispatcher 60с). **Use when:** любой вопрос «что с задачей X / что в работе».
- **Privacy:** body карточек может содержать рабочие детали — не публиковать сырьём.

### Telegram routing (`references/telegram-routing.md`)
- **Покрывает:** активный чат/топики/оператора, карту досок ↔ топиков.
- **Trust:** HIGH для маршрутизации. **Freshness:** менять только при изменении чат-структуры.
- **Privacy:** chat_id/user_id — instance-данные, не в публичные материалы (см. `leak_scan.sh`).

### Live state (инструменты, не файлы)
- **Источник:** `systemctl`, `hermes ... gateway/kanban/cron`, чтение БД в `mode=ro`.
- **Trust:** HIGHEST для вопроса «как сейчас». Бьёт любые записанные предположения.
- **Use when:** «живо ли», «сколько», «упало ли», лимиты, статусы. **Freshness:** сейчас.

### GitHub `hermes-stack` (github.com/XAPOHbI4/hermes-stack)
- **Покрывает:** версионируемый слой (контракты, скрипты, units, кроны-манифест) для развёртывания.
- **НЕ покрывает:** секреты, live state, тела скиллов — их там нет по дизайну.
- **Trust:** HIGH для «как должно быть настроено». **Privacy:** перед пушем — `leak_scan.sh` (PASS обязателен).

### Notion (источник правды Джарвиса / personalassistant)
- **Покрывает:** личные задачи, план, ожидания, календарь владельца.
- **Trust:** HIGH для личного контура; **НЕ смешивать с рабочими (Kanban) контурами.**
- **Freshness:** проверять при каждом запуске (кроны Джарвиса 9:30/21:30). **Privacy:** личные данные — обфускация при показе наружу.

### Profile memory
- **Покрывает:** стабильные предпочтения/правила. **НЕ покрывает:** прогресс задач, сырые логи, секреты, факты о текущем состоянии.
- **Trust:** MEDIUM — recall, не истина о «сейчас». **Freshness:** консолидируется кроном (≤3500 симв).

## Правило выбора
1. Вопрос про **состояние сейчас** → live check (инструменты).
2. Про **задачу** → Kanban.
3. Про **личное** → Notion.
4. Про **как-должно-быть/правила** → AGENTS.md + references + GitHub.
5. Конфликт → trust pyramid, сверху вниз. Не уверен в источнике — скажи об этом, не выдумывай.
