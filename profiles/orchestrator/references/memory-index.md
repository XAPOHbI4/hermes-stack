# Local memory index — семантический поиск по знаниям профиля

Локальный retrieval-слой поверх `references/` + `wiki/` оркестратора. Offline (fastembed,
`paraphrase-multilingual-MiniLM-L12-v2`, 384d), SQLite+FTS5, гибрид vector+BM25.
**Это навигация по знаниям, НЕ источник истины** (см. `source-map.md` — память/индекс не бьют live state).

## Когда вызывать
Когда задача зависит от локальных знаний проекта — архитектуры, процедур, ролей, политик,
ранее зафиксированного контекста — и неочевидно, какой именно reference открыть. Сначала поиск,
потом ответ.

## Как
```bash
bash /root/hermes/runtime/bin/mi_search.sh "<запрос своими словами>" [top_k]
```
Возвращает JSON: `results[] = {path, heading, score, snippet}`. Открой найденный `path` как
источник и работай с ним.

## Правила ответа (§13 брифа local-embedding)
1. Индекс — поисковый слой ПОВЕРХ документов, не канон. Открывай и цитируй сам `source`, а не snippet.
2. Не выдавай найденный фрагмент за абсолютную истину, если он старый/противоречивый — сверяйся с
   live state и `source-map.md`.
3. Если поиск ничего релевантного не вернул — **скажи об этом и запроси недостающий контекст, не выдумывай**.
4. В source of truth ничего не пишется автоматически из индекса.

## Две памяти (не путать)
- **`mi_search.sh`** — устройство СИСТЕМЫ: references/wiki оркестратора (контракты, архитектура, решения). Вопрос «как у нас устроено / какой reference открыть».
- **`kb_search.sh`** — общая БАЗА ЗНАНИЙ команды: материалы, курсы, уроки (`/root/hermes/knowledge`, обновляется ежедневно кроном). Вопрос «факты/материалы по теме».

Оба — слой навигации по источникам, НЕ истина. Правила ответа выше (цитируй source; нет релевантного — скажи) применяются к обоим.

## Сопровождение
- Пересобирается еженедельно (`memory-index.timer`, вс 09:40) и вручную:
  `cd /root/hermes/runtime/memory-index && .venv/bin/python memory_index.py rebuild-atomic`.
- Секреты в индекс не попадают: secret-gate при сборке (rebuild падает, если найден hard-secret).
- `verify` — что проиндексировано; `benchmark` — не деградировал ли поиск.
