# Wiki — текущая правда системы

Это **живой слой «как система устроена и почему СЕЙЧАС»** — отдельно от:
- `references/` — статичные операционные контракты (агент грузит, чтобы ДЕЙСТВОВАТЬ);
- profile memory — стабильные предпочтения (см. `references/memory-policy.md`);
- live state — реальное состояние прямо сейчас (проверяется инструментами).

Wiki описывает **текущую форму и решения**, references из неё вытекают. Если wiki спорит с live-check — побеждает live-check (см. trust-pyramid в `references/source-map.md`).

## Карта
- [`architecture.md`](architecture.md) — как система собрана сейчас (контур, провайдеры, контуры наблюдения/обучения).
- [`decisions.md`](decisions.md) — ключевые решения и почему так (с обоснованием).

## Правило обновления
Меняешь поведение системы → обнови `decisions.md` (что и почему). Меняешь структуру → `architecture.md`. Не дублируй сюда контракты из `references/` — здесь только «текущее состояние решений».
